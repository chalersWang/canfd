# 第一章 文档概述

## 1.1 IP 简介

CAN FD v3.0 LogiCORE™ IP 是 AMD 提供的 CAN 控制器知识产权核（IP Core），适用于 Vivado 设计套件。该 IP 核主要面向汽车电子和工业控制应用场景，包括但不限于：

- 车身控制单元（BCM）
- 测试与测量设备
- 传感器控制网络
- 工业自动化网络

该 IP 核可独立运行，也可与 MicroBlaze™ 软核处理器、Zynq™-7000 / Zynq UltraScale+™ MPSoC 中的 Arm® Cortex®-A9 处理器配合使用。

## 1.2 主要特性

1. **协议合规：** 符合 ISO 11898-1:2015 道路车辆 — 控制器局域网络（CAN）数据链路层与物理信令规范
2. **双协议支持：** 同时支持经典 CAN（CAN 2.0）和 CAN FD（CAN with Flexible Data-rate）帧格式
3. **扩展数据字段：** CAN FD 帧支持最多 64 字节数据字段（经典 CAN 为 0-8 字节）
4. **高速率支持：**
   - 快速数据速率（Data Phase）：最高 8 Mb/s
   - 标称数据速率（Nominal/Arbitration Phase）：最高 1 Mb/s
5. **发送器延迟补偿：** 支持 0-3 个数据位宽度的发送器延迟补偿（Transmitter Delay Compensation）
6. **可配置缓冲器深度：**
   - 发送（TX）邮箱缓冲器
   - 接收（RX）缓冲器，支持 FIFO（先进先出）和 Mailbox（邮箱）两种模式
7. **双 RX FIFO：** 两个 64 深度的 RX FIFO，各配备 32 组 ID 过滤器-掩码对
8. **最低 ID 优先仲裁：** 发送优先级基于 CAN ID，支持 TX 消息取消（Transmit Cancellation）
9. **独立错误记录：** 快速数据速率的独立错误记录
10. **时间戳功能：** 16 位发送时间戳计数器，32 深度发送事件 FIFO
11. **多种工作模式：**
    - 禁用自动重传模式（DAR, Disable Automatic Retransmission）
    - 总线监听模式（Snoop Mode）
    - 睡眠模式（带总线唤醒中断）
    - 内部环回模式（Internal Loopback）
    - 总线关闭恢复（自动恢复或用户干预恢复）
    - 禁用协议异常事件（DPEE）

> ⚠️ **重要许可声明：** 销售包含该 IP 核的设备前，必须获得博世（Bosch）CAN FD 协议许可证。同时需要 AMD 的 Core License Agreement 以生成硬件或比特流。

# 第二章 架构设计

## 2.1 分层架构

IP 核采用分层设计，分为两个独立运行的层：

```
+===============================================+
|              对象层 (Object Layer / LLC)         |
|              (AXI/APB 时钟域)                    |
|                                                |
|  +------------+  +-------------+  +----------+  |
|  | 寄存器模块  |  | TBMM        |  | RBMM      | |
|  | (Register  |  | (TX 缓冲器   |  | (RX 缓冲器 | |
|  |  Module)   |  |  管理模块)   |  |  管理模块) | |
|  +------------+  +-------------+  +----------+  |
|                                                |
+===============================================+
|              时钟域交叉同步器 (CDC Synchronizer)   |
+===============================================+
|              传输层 (Transfer Layer / MAC)        |
|              (CAN 时钟域)                         |
|                                                |
|  +------------------------------------------+  |
|  |        CAN FD 协议引擎 (Protocol Engine)    |  |
|  |  • 位时序控制 (Bit Timing)                  |  |
|  |  • 帧串行化/解串 (Serialization/Deserial.)  |  |
|  |  • 位填充/去填充 (Bit Stuffing/Destuffing)  |  |
|  |  • CRC 计算与校验                           |  |
|  |  • 错误检测与处理 (Error Detection)          |  |
|  |  • 过载条件识别 (Overload)                   |  |
|  +------------------------------------------+  |
|                                                |
+===============================================+
                |
         外部 CAN PHY 收发器
                |
         CAN 总线 (CAN_H / CAN_L)
```

### 对象层（Object Layer / Logical Link Control, LLC）

工作于 AXI/APB 时钟域，负责与主机处理器的通信。包含三个主要模块：

- **寄存器模块（Register Module）：** 通过 AXI4-Lite 或 APB 接口与主机通信，管理所有控制和状态寄存器
- **TX 缓冲器管理模块（TBMM, TX Buffer Management Module）：** 管理 TX 消息缓冲区的写入、仲裁和发送调度
- **RX 缓冲器管理模块（RBMM, RX Buffer Management Module）：** 管理 RX 消息的接收、过滤、缓冲和状态上报

### 传输层（Transfer Layer / Media Access Control, MAC）

工作于 CAN 时钟域，负责 CAN 总线协议的实际执行：

- 位时序的精确控制（硬同步与重同步）
- 帧的串行化发送和接收解串
- 位填充与去填充（每 5 个连续相同位插入/移除一个反相位）
- CRC（循环冗余校验）计算与校验
- 错误检测：位错误、填充错误、CRC 错误、格式错误、ACK 错误
- 错误帧和过载帧的生成与识别
- 总线关闭状态的进入与恢复

# 第三章 寄存器映射

## 3.1 地址空间分配 (32KB)

| 起始地址 | 结束地址 | 区域名称 | 存储实现 | 说明 |
|----------|----------|----------|----------|------|
| 0x0000 | 0x00FF | 核心寄存器空间 | 触发器 (Flip-Flop) | 控制和状态寄存器 |
| 0x0100 | 0x1FFF | TX 消息空间 | TX Block RAM | TX 缓冲器 + FIFO 模式 ID 过滤器 |
| 0x2000 | 0x7FFF | RX 消息空间 | RX Block RAM | RX FIFO / Mailbox + TX Event FIFO |

## 3.2 核心寄存器详细描述

### 3.2.1 软件复位寄存器 (SRR, Software Reset Register) — 0x0000

| 位 | 名称 | 访问 | 默认值 | 说明 |
|----|------|------|--------|------|
| 1 | CEN | R/W | 0 | CAN 使能位。0 = 配置模式（可修改时序/模式寄存器）；1 = 进入工作模式 |
| 0 | SRST | R/W | 0 | 软件复位。写 1 将所有配置寄存器复位为默认值。读取始终返回 0。该位在复位后自动清零 |

> ⚠️ **设计约束：** 硬件复位或软件复位后，必须等待至少 16 个 AXI/APB 时钟周期再访问任何寄存器。

### 3.2.2 模式选择寄存器 (MSR, Mode Select Register) — 0x0004

| 位 | 名称 | 访问 | 默认值 | 说明 |
|----|------|------|--------|------|
| [15:8] | ITO | R/W | 0x00 | 内部时序优化（Internal Timing Optimization）。当 BRP = 1（寄存器值为 0）时，必须设为 0x08；否则必须设为 0x00 |
| 7 | ABR | R/W | 0 | 自动总线关闭恢复请求。1 = 使能自动恢复（128 次 11 个连续隐性位） |
| 6 | SBR | R/W | 0 | 手动总线关闭恢复。仅当节点处于 Bus-Off 状态时可写；完成后硬件自动清除 |
| 5 | DPEE | R/W | 0 | 禁用协议异常事件（Disable Protocol Exception Event）。1 = 检测到 FD 帧保留位为 1 时产生 Form Error；0 = 进入 Bus Integration 状态等待总线空闲 |
| 4 | DAR | R/W | 0 | 禁用自动重传（Disable Automatic Retransmission）。1 = 每个报文仅发送一次；0 = 使能自动重传 |
| 3 | BRSD | R/W | 0 | 位速率切换禁用（Bit Rate Switch Disable）。1 = 强制以标称速率发送 CAN FD 帧（忽略 BRS 位）；0 = 遵循 TX 消息中的 BRS 位 |
| 2 | SNOOP | R/W | 0 | 总线监听模式请求。1 = 只收不发（不产生 ACK，错误计数器清零） |
| 1 | LBACK | R/W | 0 | 内部环回模式请求。1 = TX 数据内部回环至 RX，总线输出隐性 |
| 0 | SLEEP | R/W | 0 | 睡眠模式请求。1 = 核心停止活动；唤醒后硬件自动清除 |

> ⚠️ **互斥约束：** LBACK、SLEEP、SNOOP 不能同时为 1。监听模式需在复位后立即设置。

### 3.2.3 仲裁段波特率预分频器 (BRPR, Baud Rate Prescaler Register — Arbitration) — 0x0008

| 位 | 名称 | 访问 | 默认值 | 说明 |
|----|------|------|--------|------|
| [7:0] | BRP | R/W | 0x00 | 仲裁段波特率预分频值。实际分频比 = BRP + 1。仅在 CEN=0 时可写 |

### 3.2.4 仲裁段位定时寄存器 (BTR, Bit Timing Register — Arbitration) — 0x000C

| 位 | 名称 | 访问 | 默认值 | 说明 |
|----|------|------|--------|------|
| [22:16] | SJW | R/W | 0x00 | 同步跳转宽度（Synchronization Jump Width）。实际值 = SJW + 1 |
| [14:8] | TS2 | R/W | 0x00 | 时间段 2（Time Segment 2 / Phase Segment 2）。实际值 = TS2 + 1 |
| [7:0] | TS1 | R/W | 0x00 | 时间段 1（Time Segment 1 = Propagation Segment + Phase Segment 1）。实际值 = TS1 + 1 |

**标称波特率计算公式：**

```
时间量子 Tq = 1 / (CAN_CLK / (BRP + 1))
标称位宽   = Tq × (1 + (TS1 + 1) + (TS2 + 1))
标称波特率 = CAN_CLK / ((BRP + 1) × (1 + (TS1 + 1) + (TS2 + 1)))
```

**约束条件：**
- TS2 ≥ SJW（时间段 2 必须大于等于同步跳转宽度）
- 采样点 = (1 + (TS1 + 1)) / (1 + (TS1 + 1) + (TS2 + 1))

### 3.2.5 错误计数寄存器 (ECR, Error Counter Register) — 0x0010

| 位 | 名称 | 访问 | 默认值 | 说明 |
|----|------|------|--------|------|
| [15:8] | REC | R | 0x00 | 接收错误计数（Receive Error Counter）。范围 0-255 |
| [7:0] | TEC | R | 0x00 | 发送错误计数（Transmit Error Counter）。范围 0-255 |

**错误状态定义：**
- TEC ≤ 127 且 REC ≤ 127 → 错误主动状态（Error Active）
- TEC ≥ 128 或 REC ≥ 128 → 错误被动状态（Error Passive）
- TEC ≥ 256 → 总线关闭状态（Bus-Off）

> 在监听模式（Snoop Mode）下，REC 和 TEC 均被清零。

### 3.2.6 错误状态寄存器 (ESR, Error Status Register) — 0x0014

| 位 | 名称 | 访问 | 默认值 | 说明 |
|----|------|------|--------|------|
| 11 | F_BERR | R/W1C | 0 | 快速数据相位位错误（FD Bit Error） |
| 10 | F_STER | R/W1C | 0 | 快速数据相位填充错误（FD Stuff Error） |
| 9 | F_FMER | R/W1C | 0 | 快速数据相位格式错误（FD Form Error） |
| 8 | STER | R/W1C | 0 | 填充错误（Stuff Error） |
| 7 | FMER | R/W1C | 0 | 格式错误（Form Error） |
| 6 | CRCER | R/W1C | 0 | CRC 错误（CRC Error） |
| 5 | ACKER | R/W1C | 0 | 应答错误（ACK Error） |
| 4 | BERR | R/W1C | 0 | 位错误（Bit Error） |

> R/W1C = 可读/写 1 清除（Write-1-to-Clear）

### 3.2.7 中断寄存器组

#### 中断状态寄存器 (ISR, Interrupt Status Register) — 0x0018

| 位 | 名称 | 默认值 | 说明 |
|----|------|--------|------|
| 10 | RXMNF | 0 | RX Mailbox Not Full — Mailbox 模式：有缓冲器就绪可接收新消息 |
| 9 | TXE_WM | 0 | TX Event FIFO Watermark — TX 事件 FIFO 达到水印 |
| 8 | TXE_FLL | 0 | TX Event FIFO Full — TX 事件 FIFO 满 |
| 7 | TOVF | 0 | Timestamp Overflow — 时间戳计数器溢出 |
| 6 | TAPF | 0 | TX Arbitration Process Finished — 一次发送仲裁过程完成 |
| 5 | RXFWMFLL | 0 | RX FIFO Watermark/Full |
| 4 | RXOVF | 0 | RX FIFO Overflow — RX FIFO 溢出 |
| 3 | RXOK | 0 | RX Message Received — 成功接收报文 |
| 2 | TXOK | 0 | TX Message Transmitted — 成功发送报文 |
| 1 | WKUP | 0 | Wake-Up — 总线唤醒事件 |
| 0 | EWARN | 0 | Error Warning — 错误警告 |

#### 中断使能寄存器 (IER, Interrupt Enable Register) — 0x0020

- 位定义与 ISR 一一对应
- 写 1 = 使能对应中断
- 写 0 = 屏蔽对应中断

#### 中断清除寄存器 (ICR, Interrupt Clear Register) — 0x0024

- 位定义与 ISR 一一对应
- 写 1 = 清除对应中断标志
- 写 0 = 无操作

> 中断信号为高有效电平敏感信号，与 AXI/APB 时钟同步。

# 第四章 功能描述

## 4.1 工作模式详解

### 4.1.1 配置模式 (Configuration Mode)
- **进入条件：** CEN = 0
- **行为：** CAN 控制器停止所有操作。所有配置寄存器（时序、模式、过滤器）可写。总线输出隐性电平。
- **典型用途：** IP 初始化、参数配置、模式切换前准备。

### 4.1.2 正常模式 (Normal Mode)
- **进入条件：** CEN = 1, LBACK = 0, SLEEP = 0, SNOOP = 0
- **行为：**
  - 参与完整的 CAN 总线通信
  - 发送时响应 ACK/错误帧
  - 接收时产生 ACK
  - 错误计数器正常累加
  - 支持自动重传（DAR=0）或单次发送（DAR=1）

### 4.1.3 内部环回模式 (Internal Loopback Mode)
- **进入条件：** CEN = 1, LBACK = 1
- **行为：**
  - TX 数据内部回环至 RX 通路
  - 总线输出隐性电平（不驱动外部总线）
  - 可用于自测试和调试
- **约束：** 不能与 SLEEP、SNOOP 同时使能

### 4.1.4 睡眠模式 (Sleep Mode)
- **进入条件：** CEN = 1, SLEEP = 1
- **行为：**
  - 核心时钟停止，控制器进入低功耗状态
  - 可通过 CAN 总线活动唤醒
  - 唤醒后 SLEEP 位自动清除，产生 WKUP 中断
- **约束：** 不能与 LBACK、SNOOP 同时使能

### 4.1.5 总线监听模式 (Snoop Mode)
- **进入条件：** CEN = 1, SNOOP = 1（需复位后立即设置）
- **行为：**
  - 只监听总线数据，不发送任何帧（包括 ACK 和错误帧）
  - 错误计数器强制清零
  - 可用于总线监控和诊断
- **约束：** 不能与 LBACK、SLEEP 同时使能

## 4.2 发送流程

```
┌─────────────────────────────────────────────────┐
│ 1. 配置模式 (CEN=0)                               │
│    ├── 设置 BRPR, BTR (标称位时序)                │
│    ├── 设置 DP_BRPR, DP_BTR (数据相位位时序)       │
│    ├── 设置 MSR (工作模式参数)                     │
│    └── 设置 IER (使能所需中断)                     │
└──────────────────┬──────────────────────────────┘
                   ▼
┌─────────────────────────────────────────────────┐
│ 2. 启动工作 (CEN=1, 选择 Normal/Loopback 模式)     │
└──────────────────┬──────────────────────────────┘
                   ▼
┌─────────────────────────────────────────────────┐
│ 3. 填充 TX 消息缓冲器                             │
│    ├── 向 TX 消息空间 (0x0100-0x1FFF) 写入消息    │
│    ├── 设置 CAN ID, DLC, 数据字段 (0-64字节)       │
│    └── 设置帧格式标志 (IDE, RTR, FDF, BRS, ESI)    │
└──────────────────┬──────────────────────────────┘
                   ▼
┌─────────────────────────────────────────────────┐
│ 4. 请求发送                                       │
│    └── 写 TRR 对应位 = 1                          │
└──────────────────┬──────────────────────────────┘
                   ▼
┌─────────────────────────────────────────────────┐
│ 5. 仲裁与发送                                     │
│    ├── CAN 核按最低 ID 优先原则仲裁               │
│    ├── 赢得仲裁后开始发送                          │
│    ├── DAR=0: 失败后自动重传                       │
│    └── DAR=1: 单次发送，失败后不重传               │
└──────────────────┬──────────────────────────────┘
                   ▼
┌─────────────────────────────────────────────────┐
│ 6. 发送完成处理                                   │
│    ├── 产生 TXOK 中断                             │
│    ├── 写入 TX 事件 FIFO (含时间戳)                │
│    └── TRR 对应位自动清除                         │
└─────────────────────────────────────────────────┘
```

### TX 消息取消机制

- 通过 TCR 寄存器写入对应位可取消尚未开始发送的消息
- 取消成功后产生中断，TX 缓冲器被释放

## 4.4 过滤器机制

### FIFO 模式
- 32 组 ID 过滤器/掩码对
- 存放于 TX Block RAM 空间的过滤器区域
- 由 AFR（接受过滤器寄存器）统一控制使能
- 匹配逻辑：`(Received_ID & Mask) == (Filter_ID & Mask)`

### Mailbox 模式
- 每个 RX 缓冲器拥有独立的 ID 和 ID 掩码
- 存放在各自缓冲器头部
- 不需要全局 AFR 寄存器
- 每个邮箱独立匹配和接收

## 4.5 位时序与采样点

### 标称位时序段构成

```
+------+--------------------+-------------------+
| SYNC |   Time Segment 1   |   Time Segment 2  |
| SEG  |  (PROP + PHASE1)   |     (PHASE2)      |
+------+--------------------+-------------------+
| 1Tq  |   (TS1+1) × Tq    |  (TS2+1) × Tq     |
+------+--------------------+-------------------+
                 ↑ 采样点
```

### CAN FD 数据相位位时序

CAN FD 帧在数据相位阶段可以使用不同的、更高的位速率。数据相位时序参数独立配置（DP_BRPR, DP_BTR），在 BRS（Bit Rate Switch）位为隐性时生效。

### 发送器延迟补偿 (TDC, Transmitter Delay Compensation)

在 CAN FD 数据相位高速率下，发送器的环路延迟可能超过一位时间。TDCOFF 参数（0-3 数据位）定义了从发送位边沿到采样点之间的偏移补偿。

## 4.6 错误处理

### 错误类型
| 错误类型 | 检测机制 | ESR 标志 |
|----------|----------|----------|
| 位错误 (Bit Error) | 发送的位与总线上监视到的位不一致 | BERR / F_BERR |
| 填充错误 (Stuff Error) | 连续 6 个相同位检测 | STER / F_STER |
| CRC 错误 (CRC Error) | 接收方计算的 CRC 与发送方不一致 | CRCER |
| 格式错误 (Form Error) | 固定格式位（CRC分隔符、ACK分隔符、EOF）值非法 | FMER / F_FMER |
| ACK 错误 (ACK Error) | 发送方在 ACK 槽位未检测到显性位 | ACKER |

### 错误状态转换

```
     正常通信
        │
        ▼ (TEC>127 或 REC>127)
┌───────────────┐
│ 错误被动状态    │
│ (Error Passive)│
└───────┬───────┘
        │ (TEC>255)
        ▼
┌───────────────┐      128次11个连续隐性位
│ 总线关闭状态    │ ──────────────────────────► 错误主动状态
│ (Bus-Off)      │      (自动或手动恢复)
└───────────────┘
```

## 4.7 时间戳功能

- 16 位自由运行计数器（TSR），随 CAN 位时间自动递增
- 成功发送的每条消息都会记录时间戳并写入 TX 事件 FIFO
- TX 事件 FIFO 深度为 32
- 计数器溢出产生 TOVF 中断

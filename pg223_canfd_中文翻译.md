# 第1章 引言

AMD LogiCORE(TM) IP CAN 灵活数据速率（CAN FD）内核非常适用于汽车和工业应用，例如汽车车身控制单元、汽车测试设备、仪表盘、传感器控制和工业网络。该内核可独立使用，也可连接到 AMD MicroBlaze(TM) 处理器或 AMD Zynq(TM) 7000 SoC、AMD Versal(TM) 器件及 AMD UltraScale+(TM) MPSoC 中的 Arm® Cortex-A9 处理器。

## 特性

- 遵循 ISO 11898-1/2015：道路车辆 -- 控制器局域网（CAN）第1部分：数据链路层和物理信令规范设计
- 支持 CAN 和 CAN FD 帧
- 支持 ISO 11898-1/2015 规范中定义的 CAN FD 帧格式
- 支持最多 64 字节的 CAN FD 帧
- 支持最高 8 Mb/s 的灵活数据速率
- 支持最高 1 Mb/s 的标称数据速率
- 最多三个数据位发送器延迟补偿
- TX 和 RX 邮箱缓冲区，深度可配置
- 两个深度为 64 的 RX FIFO，配备 32 对 ID 滤波器-掩码对
- 最低 ID 的消息优先发送
- 支持 TX 消息取消
- 快速数据速率的独立错误记录

> **重要！** 在销售包含 CAN FD 内核的器件之前，必须持有有效的 Bosch CAN FD 协议许可证。

## 其他特性

- 已发送和已接收消息的时间戳
- 支持发送事件 FIFO
- 支持以下模式：
  - 禁用自动重传（DAR）模式
  - 监听（总线监控）模式
  - 带唤醒中断的睡眠模式
  - 内部回环模式
  - 总线关闭恢复模式
    - 自动恢复
    - 用户干预自动恢复
  - 禁用协议异常事件模式

## 内核描述

内核功能分为两个独立层，如"关于本内核"中所示。对象层通过 AXI4-Lite/APB 接口与主机控制接口交互，工作在 AXI4-Lite/APB 时钟域。传输层与外部 PHY 接口交互，工作在 CAN 时钟域。两层之间的信息交换通过 CDC 同步器完成。CAN FD 对象层提供先进的发送和接收方法来管理消息缓冲区。

### 对象层（逻辑链路层）

对象层分为以下子模块：

- **寄存器模块（Register Module）：** 该模块允许通过外部主机接口对寄存器进行读写访问。
- **TX 缓冲区管理模块（TBMM）：** TX 缓冲区管理模块与 CAN FD 协议引擎接口，提供下一个要在 CAN 总线上发送的缓冲区。它管理主机对 TX 块 RAM 的访问。
- **RX 缓冲区管理模块（RBMM）：** RX 缓冲区管理模块与 CAN FD 协议引擎接口，为从 CAN 总线接收消息提供存储。它管理主机对 RX 块 RAM 的访问。

### 传输层（介质访问控制层）

传输层提供以下主要功能：

- **识别总线空闲后启动发送过程**（遵循帧间空间）
  - 帧的串行化
  - 位填充（Bit Stuffing）
  - 仲裁（Arbitration）及仲裁失败时转入接收模式
  - ACK 检查
  - 向 PHY 提供串行比特流用于发送
  - CRC 序列计算（包括 FD 帧的填充位计数）
  - 比特率切换

- **从 PHY 接收串行比特流**
  - 解串行化与重构帧结构
  - 位去填充（Bit Destuffing）
  - ACK 发送
  - 比特率切换

- **位时序（Bit Timing）功能**
- **错误检测与信令**
- **过载条件识别与响应**

### 软件复位寄存器（SRR，地址偏移 + 0x0000）

写入软件复位寄存器（SRR）将内核置于配置模式（Configuration Mode）。在配置模式下，内核在总线线上驱动隐性电平，不发送或接收消息。上电期间，CEN 和 SRST 位为 0，状态寄存器（SR）中的 CONFIG 位为 1。传输层配置寄存器只能在 SRR 中的 CEN 位为 0 时更改。模式选择寄存器位（除 SLEEP 和 SBR 外）只能在 CEN 位为 0 时更改。如果在内核运行期间更改 CEN 位，建议复位内核以便重新开始运行。

**表4：软件复位寄存器（SRR）**

| 位 | 名称 | 访问 | 默认值 | 描述 |
|----|------|------|--------|------|
| 31:2 | 保留（Reserved） | - | 0 | 保留 |
| 1 | CEN | R/W | 0 | **CAN 使能（CAN Enable）**。内核的使能位。<br>- 1 = 内核处于回环、睡眠、监听或正常模式，取决于 MSR 中的 LBACK、SLEEP 和 SNOOP 位<br>- 0 = 内核处于配置模式<br>**注：** 如果在内核运行期间清除 CEN 位，AMD 建议复位内核以便重新开始运行 |
| 0 | SRST | WO | 0 | **复位（Reset）**。内核的软件复位位。<br>- 1 = 内核复位。如果向此位写入 1，所有内核配置寄存器（包括 SRR）都将复位。对此位的读取始终返回 0<br>- 执行软复位或硬复位后，等待 16 个 AXI4-Lite/APB 时钟周期再发起下一次 AXI4-Lite/APB 事务 |

### 模式选择寄存器（MSR，地址偏移 + 0x0004）

写入模式选择寄存器（MSR）可使内核进入监听（Snoop）、睡眠（Sleep）、回环（Loopback）或正常（Normal）模式。在正常模式下，内核参与正常的总线通信。如果 SLEEP 位设置为 1，内核进入睡眠模式。如果 LBACK 位设置为 1，内核进入回环模式。如果 SNOOP 位设置为 1，内核进入监听模式，不参与总线通信，仅接收消息。

> **重要！** 不要同时将 LBACK、SLEEP 和 SNOOP 位设置为 1。在任何给定时刻，内核只能处于回环、睡眠或监听模式之一。当所有三个位都设置为 0 时，内核可在满足其他条件的情况下进入正常模式。

**表5：模式选择寄存器（MSR）**

| 位 | 名称 | 访问 | 默认值 | 描述 |

### 错误计数器寄存器（ECR，地址偏移 + 0x0010）

ECR 为只读寄存器。对 ECR 的写操作无效。寄存器中错误计数器的值反映内核中发送和接收错误计数器的值。以下条件会复位发送和接收错误计数器：

- 向 SRR 中的 SRST 位写入 1
- 向 SRR 中的 CEN 位写入 0
- 内核进入总线关闭状态
- 在总线关闭恢复期间，直到内核进入错误主动状态（在 128 次出现 11 个连续隐性位之后）

> **重要！** 在总线关闭恢复期间，当检测到 11 个连续标称隐性位的序列时，接收错误计数器递增 1。

> **注：** 在监听模式（SNOOP）下，错误计数器被禁用并清零为 0。对错误计数器寄存器的读取返回 0。

| 15:8 | REC[7:0] | R | 0 | **接收错误计数（Receive Error Count）**。指示接收错误计数器的值 |

### 验收滤波器（控制）寄存器（AFR，地址偏移 + 0x00E0）

在 RX 顺序/FIFO 缓冲区模式下，验收滤波器（控制）寄存器（AFR）定义使用哪些验收滤波器。每个验收滤波器 ID 寄存器（AFIR）和验收滤波器掩码寄存器（AFMR）对关联一个 UAF 位。

- 当 UAF 位为 1 时，对应的验收滤波器对用于验收滤波
- 当 UAF 位为 0 时，对应的验收滤波器对不用于验收滤波
- 要在正常模式下修改验收滤波器对，此寄存器中对应的 UAF 位必须设置为 0
- 修改验收滤波器后，对应的 UAF 位必须设置为 1
- 如果所有 UAF 位设置为 0，接收的消息不存储在 RX 顺序/FIFO 缓冲区中

**表26：验收滤波器（控制）寄存器（AFR）**

| 位 | 名称 | 访问 | 默认值 | 描述 |
|----|------|------|--------|------|
| 31:0 | UAF31-UAF0 | R/W | N/A | **使用验收滤波器掩码对**。<br>- 1 = 对应的 AFMR 和 AFID 寄存器对用于验收滤波<br>- 0 = 对应的 AFMR 和 AFID 寄存器对不用于验收滤波 |

**注：** 此寄存器空间在 RX 邮箱缓冲区模式下保留。写无效，读返回 0。

## 验收滤波器

在 RX 顺序/FIFO 模式中有 32 个验收滤波器。每个验收滤波器有一个验收滤波器掩码寄存器（AFMR）和一个验收滤波器 ID 寄存器（AFIR），由验收滤波器（控制）寄存器位控制。

### 验收滤波（RX FIFO-1 不存在或禁用时）

按以下顺序执行验收滤波：
1. 传入的标识符与验收滤波器掩码寄存器中的位进行掩码运算
2. 验收滤波器 ID 寄存器也与掩码寄存器中的位进行掩码运算
3. 比较两个结果值
4. 如果相等，消息存储在 RX FIFO-0 中
5. 由每个已定义的滤波器处理。如果传入标识符通过任何验收滤波器，消息存储在 RX FIFO-0 中

> **注：** 可通过编程 RXFP 为 'd31（在 RX FIFO 水位线寄存器中）来禁用 RX FIFO-1。

### 验收滤波（RX FIFO-1 使能时）

在此情况下，RXFP 字段（在 RX FIFO 水位线寄存器中）需要小于 'd31。ID 匹配过程是顺序过程，从最低使能的滤波器开始，在第一个匹配处停止。

CAN FD 支持以下模式和状态：配置（Configuration）、正常（Normal）、回环（Loopback）、睡眠（Sleep）、监听（Snoop）、协议异常（PEE）状态、总线关闭恢复状态。

---

### 配置模式

当执行以下任一操作时，内核进入配置模式（无论当前处于何种工作模式）：
- 向 SRR 寄存器中的 CEN 位写入 0
- 向 SRR 寄存器中的 SRST 位写入 1
- 在复位输入上驱动 0

复位后，在设置 CEN 位且在 CAN 总线上看到 11 个连续标称隐性位之后，内核退出配置模式。

配置模式特性：
- 控制器失去与 CAN 总线的同步，在 TX 线上驱动恒定的隐性位
- 错误计数器寄存器复位
- 错误状态寄存器复位
- BTR 和 BRPR 寄存器可修改
- 状态寄存器中 CONFIG 位为 1
- 内核不接收任何新消息、不发送任何消息
- 所有配置寄存器可访问
- 待发送消息被保留，恢复正常操作后发送
- 消息取消被允许
- 中断状态寄存器位 ARBLST、TXOK、RXOK、RXOFLW、RXOFLW_1、ERROR、BSOFF、SLP 和 WKUP 被清除

### 正常模式

> **重要！** 内核仅在 MSR 寄存器中 Sleep、Loopback 和 Snoop 位均为 0 时才能进入正常模式。

在正常模式下，内核通过发送和接收消息参与总线通信。

> **注：** 在正常模式下，内核不存储自己发送的消息。

### 回环模式（内部）

> **重要！** 此模式用于诊断目的。

在回环模式下，内核通过内部回环到 RX 线来接收自己发送的任何消息并应答。不参与正常总线通信，不接收其他 CAN 节点发送的消息（忽略外部 TX 线）。在 CAN 总线上驱动隐性比特流（外部 TX 线）。

### 睡眠模式

当 SLEEP 位为 1、CAN 总线空闲且无待处理发送请求时，内核从配置模式或正常模式进入睡眠模式。在以下唤醒条件下进入正常模式：
- SLEEP 位被设置为 0
- SLEEP 位为 1 且检测到总线活动
- 有新的待发送消息

### 监听（总线监控）模式

> **重要！** 此模式用于诊断目的。

- 内核在 CAN 总线上发送隐性位
- 内核接收其他节点发送的消息但不发送 ACK
- 错误计数器被禁用并清零为 0

> **注：** 建议仅在系统复位或软件复位后编程监听模式。

### 协议异常状态

如果接收到 CAN FD 帧中 res 位为隐性（前提是 MSR 寄存器中 DPEE 位未设置），CAN FD 内核进入 PEE 状态。在检测到 11 个连续标称隐性位后退出此状态。

当 CAN FD 节点检测到 PEE 时，软件需要执行以下步骤恢复发送：
1. 读取 TX 缓冲区就绪请求（TRR）值
2. 将 TRR 读取值写入 TX 缓冲区取消请求（TCR）
3. 等待 TRR 变为 0
4. 将步骤 1 中读取的值写入 TRR

### 总线关闭恢复状态

如果发送错误计数达到或超过其端点，CAN FD 进入总线关闭状态。恢复由 MSR 中的自动恢复（ABR）或手动恢复（SBR）位控制。

恢复步骤与 PEE 相同。

## 编程模型

本节介绍编程 CAN FD 进行运行所需的各种配置步骤：

### 寄存器配置顺序

1. **选择工作模式**：Normal/Sleep/Loopback/Snoop----相应设置 MSR 寄存器中的 LBACK、SNOOP、SLEEP 位
2. **配置传输层配置寄存器**：
   > **重要！** 确保网络中所有 CAN FD 节点编程为具有相同的仲裁相位位速率、数据相位位速率、仲裁相位采样点位置和数据相位采样点位置。
   - 编程 BRPR 和 BTR 寄存器
   - 编程 DP_BRPR 和 DP_BTR 寄存器（含 TDC 控制字段）
3. **配置验收滤波器寄存器**（仅接收顺序/FIFO 模式）
4. **配置掩码寄存器**（仅接收邮箱模式）
5. **编程中断使能寄存器**
6. **通过向 SRR 中的 CEN 写入 1 使能协议控制器**

### 消息发送

**主机操作：**
1. 轮询 TRR 寄存器检查待处理发送请求
2. 如果有未设置/清除的 TRR 位，可添加新发送请求
3. 在 TX 块 RAM 中准备消息元素（写入有效 ID、DLC、DW 字段）
4. 使能所需中断生成
5. 设置对应 TRR 位以启用缓冲区就绪请求
6. 等待中断或轮询 TRR 获取请求状态

**内核操作：**
1. 确定下一个最高优先级缓冲区发送。如果两个缓冲区具有相同 ID，选择索引较低的
2. 如果使能，复制 ID 和 DLC 字段到 TX Event FIFO 并添加时间戳和事件类型
3. 发送请求被服务后清除相应的 TRR 位

### 发送取消

主机操作与发送类似，但通过 TCR 寄存器设置取消请求位。如果缓冲区已被传输层锁定发送，取消在发送周期结束时执行。

### 接收（顺序缓冲区/FIFO 模式）

**主机操作：**
1. 编程 RX FIFO 水位线寄存器
2. 使能中断
3. 通过轮询 FSR 或水位线满中断获知新消息
4. 从读索引位置读取消息
5. 通过设置 IRI 位递增读索引

### 接收（邮箱模式）

**主机操作：**
1. 准备接收缓冲区（写入 ID 和掩码，将缓冲区状态从非活跃变为活跃）
2. 使能中断
3. 从状态为"已满"的缓冲区读取消息
4. 读取后可变更状态回活跃或非活跃

## 时钟

CAN FD 有两个时钟：CAN 时钟和 AXI4-Lite/APB 时钟。这两个时钟可异步或同步。异步时，AXI4-Lite/APB 时钟频率需大于 CAN 时钟频率。

- **CAN 时钟频率：** 8 至 80 MHz
- **AXI4-Lite/APB 时钟频率：** 8 至 200 MHz

内核还有第三个时钟 can_clk_x2，与 can_clk 完全同步，频率为其 2 倍。

> **重要！** CAN 时钟必须符合相关标准中给出的振荡器容差范围。

## 复位

CAN FD 可通过系统（硬件）复位输入端口或软件控制复位（SRR 中的 SRST 位）进行复位。传输层在 CEN 位为 0 期间保持复位状态。

- **对象层：** 相对于 AXI4-Lite/APB 时钟同步复位
- **传输层：** 异步复位断言，同步复位取消断言

> **重要！** 施加任何复位时，TX 块 RAM 和 RX 块 RAM 的内容不会被清除。

## 中断

内核有一个电平敏感中断输出 ip2bus_intrevent。

中断线断言的两个条件：
1. ISR 中某位为 1 且 IER 中对应位为 1
2. IER 位从 0 变为 1 时 ISR 中对应位已为 1

中断线取消断言的两个条件：
1. 清除 ISR 中的 1 位（通过向 ICR 对应位写入 1，前提是 IER 对应位为 1）
2. IER 位从 1 变为 0 时 ISR 中对应位为 1

同时发生断言和取消断言时，中断线先取消断言，若断言条件仍为 TRUE 则重新断言。

# 附录A 调试

## 常见硬件调试问题

1. **TX/RX 线上看不到期望的波特率。** 确保期望值已写入 BRPR 和 BTR 寄存器。实际值为寄存器值加一。
2. **使能后内核未达到 CONFIG 状态。** 确保内核检测到 11 个连续隐性位。
3. **内核已使能且正确写入 BRPR/BTR，但线路未翻转。** 确保 can_clk 端口已连接。
4. **CAN FD 内核在与其他节点的网络中产生各种帧错误。** 确保所有节点具有相同的仲裁/数据相位位速率和采样点位置。

## AXI4-Lite 接口调试

> **图11：AXI4-Lite 接口读时序图**（详见原文第95页）

如果接口无响应，确保：
- s_axi_aclk 和输入时钟已连接并翻转
- 接口未处于复位状态
- s_axi_aclken 为高电平有效
- 主内核时钟正在翻转，使能信号已断言

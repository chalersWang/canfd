## 一、IP 概述

| 项目 | 内容 |
|------|------|
| IP 名称 | CAN FD v3.0 LogiCORE IP |
| 发布者 | AMD (Xilinx) |
| 协议标准 | ISO 11898-1:2015 |
| 目标器件 | Versal, UltraScale+, UltraScale, Zynq-7000, 7 Series, Zynq UltraScale+ MPSoC |
| 主机接口 | AXI4-Lite（默认）或 APB（可选） |
| 典型应用 | 汽车车身控制、工业网络、传感器控制、测试设备 |

## 二、关键规格参数

| 参数 | 值 |
|------|-----|
| 标称数据速率 (Nominal) | 最高 1 Mb/s |
| 快速数据速率 (FD) | 最高 8 Mb/s |
| CAN 2.0 数据字段 | 0-8 字节 |
| CAN FD 数据字段 | 0-64 字节 |
| TX 缓冲器 | 最多 32 个 (Mailbox模式) |
| RX FIFO 深度 | 2 个 FIFO，各 64 深度 |
| RX 缓冲器 (Mailbox) | 最多 48 个 |
| ID 过滤器 | FIFO模式：32组ID/Mask；Mailbox：每缓冲器独立 |
| TX 事件 FIFO | 32 深度 |
| 时间戳 | 16 位计数器 (TSR) |
| 发送延迟补偿 | 0-3 数据位 (TDCOFF) |
| 中断 | 单条电平敏感中断线 |

## 五、核心寄存器速查

### 5.1 软件复位 (SRR, 0x0000)
| Bit | 名称 | 说明 |
|-----|------|------|
| 1 | CEN | CAN使能。0=配置模式; 1=工作模式 |
| 0 | SRST | 软件复位。写1复位全部配置寄存器 |

> ⚠️ 复位后需等待至少16个AXI/APB时钟周期

### 5.2 模式选择 (MSR, 0x0004)
| Bits | 名称 | 说明 |
|------|------|------|
| [15:8] | ITO | BRP=1时必须=0x08，否则=0x00 |
| 7 | ABR | 自动总线关闭恢复 |
| 6 | SBR | 手动启动总线关闭恢复 |
| 5 | DPEE | 禁用协议异常事件(1=Form Error) |
| 4 | DAR | 禁用自动重传(1=单次发送) |
| 3 | BRSD | 位速率切换禁用(1=强制标称速率) |
| 2 | SNOOP | 监听模式 |
| 1 | LBACK | 环回模式 |
| 0 | SLEEP | 睡眠模式 |

> ⚠️ LBACK/SLEEP/SNOOP 不可同时置1

### 5.3 位时序 (BRPR 0x0008 / BTR 0x000C)
- **BRPR**: BRP[7:0] — 分频值 (实际 = BRP+1)
- **BTR**: SJW[22:16], TS2[14:8], TS1[7:0] (实际值均为寄存器值+1)

```
标称波特率 = CAN_CLK / ((BRP+1) × (1 + (TS1+1) + (TS2+1)))
```

### 5.4 数据相位位时序 (DP_BRPR 0x0088 / DP_BTR 0x008C)
- **DP_BRPR**: DP_BRP[7:0]
- **DP_BTR**: DP_SJW, DP_TS2, DP_TS1, **TDCOFF** (发送延迟补偿 0-3位)

### 5.5 错误计数器 (ECR 0x0010, 只读)
| Bits | 名称 | 说明 |
|------|------|------|
| [15:8] | REC | 接收错误计数 |
| [7:0] | TEC | 发送错误计数 |

### 5.6 错误状态 (ESR 0x0014, 写1清除)
| Bit | 标志 | 说明 |
|-----|------|------|
| 11 | F_BERR | 快速数据相位位错误 |
| 10 | F_STER | 快速数据相位填充错误 |
| 9 | F_FMER | 快速数据相位格式错误 |
| 8 | STER | 填充错误 |
| 7 | FMER | 格式错误 |
| 6 | CRCER | CRC错误 |
| 5 | ACKER | ACK错误 |
| 4 | BERR | 位错误 |

### 5.7 中断寄存器组
| 寄存器 | 地址 | 功能 |
|--------|------|------|
| ISR | 0x0018 | 中断状态 (读) |
| IER | 0x0020 | 中断使能 (读写) |
| ICR | 0x0024 | 中断清除 (写1清除) |

### 5.8 其他重要寄存器
| 寄存器 | 地址 | 说明 |
|--------|------|------|
| SR | 0x0018 | 状态寄存器 |
| TSR | 0x0028 | 16位时间戳计数器 |
| TRR | 0x0090 | TX缓冲器就绪请求 (32bit, 每bit对应一个缓冲器) |
| TCR | 0x0098 | TX缓冲器取消请求 |
| TxE_FSR | 0x00A0 | TX事件FIFO状态 |
| TxE_WMR | 0x00A4 | TX事件FIFO水印 |
| RCS0-2 | 0x00B0-B8 | RX Mailbox控制/状态 |
| AFR | 0x00E0 | 接受过滤器控制 (FIFO模式) |
| FSR | 0x00E8 | RX FIFO状态 |
| WMR | 0x00EC | RX FIFO水印 |

## 六、工作模式

| 模式 | CEN | 配置 | 行为 |
|------|-----|------|------|
| **配置模式** | 0 | — | 所有寄存器可写，输出隐性 |
| **正常模式** | 1 | LBACK=SLEEP=SNOOP=0 | 正常收发，ACK响应 |
| **环回模式** | 1 | LBACK=1 | 内部自发自收，总线输出隐性 |
| **睡眠模式** | 1 | SLEEP=1 | 停止活动，总线唤醒后自动清零 |
| **监听模式** | 1 | SNOOP=1 | 只收不发，错误计数器清零 |

### 总线关闭恢复
- 自动恢复：MSR.ABR=1 → 检测128次连续11个隐性位后恢复
- 手动恢复：MSR.SBR=1 → 用户主动触发恢复

## 七、TX/RX 操作流程

### 发送流程
```
1. 配置模式(CEN=0) 设置时序/模式参数
2. CEN=1 进入工作模式
3. 写入 TX 消息到 TX 缓冲器空间 (0x0100-0x1FFF)
4. 置 TRR 对应位为1 → 请求发送
5. 核按最低ID优先仲裁 → 发送
6. 发送完成/失败 → 产生中断 → 可选写入TX事件FIFO
7. DAR=1: 单次发送; DAR=0: 自动重传
```

### 接收流程
```
1. 配置过滤器(ID/Mask)
2. 匹配报文 → 存入RX缓冲器
3. FIFO模式: → RX FIFO → 水印/满中断
4. Mailbox模式: → 对应缓冲器 → 满标志
5. 主机读取RX消息空间 → 释放缓冲器
```

---

## 八、过滤器配置

### FIFO 模式
- 32组 ID过滤器/Mask对 → 存于TX Block RAM过滤器区域
- 由 AFR（接受过滤器寄存器）控制使能

### Mailbox 模式  
- 每个RX缓冲器独立ID和ID Mask → 存于缓冲器头部
- 无需全局AFR

---

## 九、重要设计注意事项

1. **许可证**：终端产品必须持有博世CAN FD协议许可证
2. **复位后延迟**：软/硬复位后必须等16个AXI/APB时钟周期
3. **互斥模式**：LBACK/SLEEP/SNOOP禁止同时置1
4. **ITO字段**：BRP=1时必须设MSR.ITO=0x08
5. **CEN清除**：工作中清除CEN后建议复位再重启
6. **AXI4-Lite**：不支持wstrb信号
7. **外部PHY**：必须外接CAN收发器
8. **中断**：电平敏感，高有效
